📋 Chart Refactor — Per-Ticker JSON API + Native React lightweight-charts

📅 2026-06-24 📁 breakout-signal ⏱️ Effort: ~1 day Risk: MEDIUM RISK Status: PLANNED
Scope: chart data + render refactor only. The wider "Breakout Terminal" redesign (slim nav rail, 4 tabs, FAQ drawer, new color tokens) is a separate plan. This one swaps the monolithic generated chart.html + Postgres mega-blob for a small per-ticker JSON store and a native React lightweight-charts panel. The data-builder get_chart_data() stays as-is.

Note: GET /api/chart/{ticker} already exists in scan.py but returns only bare OHLCV (recomputed, no EMA/SMA/signals/markers). This plan replaces its body to serve the rich dict.

🔄 Section 5 — Rollback Plan

⚠️ If something breaks

1. Revert the commit: git revert HEAD (changes are additive — old combined-HTML + iframe path returns intact).

2. Redeploy previous Railway build.

3. The new chart_data table is harmless if unused; no need to drop it. Optionally DROP TABLE chart_data; later.

4. No data migration was performed — the old DailyState 'chart_html' row is left untouched and will be re-populated by the reverted main.py on the next EOD scan.

5. If only the frontend panel is broken: temporarily restore the <iframe src="/chart"> in bs-views.jsx (backend API can stay).
